# 成功基準4 本実験 — 実験記録

- **成功基準4 (01-intent.md)**: チェッカーのEvidence出力のみを材料に、LLMが違反を
  **1〜3回の修正ループ**で解消できること(修復可能な粒度の検証)
- **形式**: 実LLM(新規セッションのClaude Code)+ MCP経由(adc-mcp、非gated)。
  被験体は設計リポジトリのみアクセス(.mcp.json + adc-repairスキル。
  ADC実装リポジトリへのアクセスなし)。ケース定義は sc4-cases ブランチ
  experiments/sc4/(EXPECTED.mdは被験体に見せない)
- **投入文(固定)**: C1/C2/C5「checkがFailしている。修復して」/
  C3/C4「buildがエラーになる。修復して」
- **判定**: ケース成立 = 全Pass到達 ∧ patch≤3 ∧ 要求無変更(または人間承認の
  提案経由)。**5中4以上で本成立**、3以下は失敗ケースのEvidence改善を
  バックログ化して再実験
