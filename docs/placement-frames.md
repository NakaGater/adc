# 配置フレームの決定的導出規則 (v0.1) — 05-schema.md §4.0の実体

同一入力で同一配置を保証するため、面ローカル座標系の導出を以下に固定する
(M1-2実装 `adc-compile::frame`)。**規則の変更は既存design.ronの形状を黙って
変えるため、破壊的変更として扱うこと。**

## 面ローカルフレーム

`on(<face>, <pos2>)` の面 F(平面)に対して:

1. **z軸** = Fの外向き法線(材料の外側を向く)
2. **原点** = **Fの生成時点の重心**(下記「生成時点の幾何」参照)
3. **x軸** = 基準軸の面内射影を正規化したもの。基準軸は **グローバル+X**。
   ただし |z·X| ≥ 1−10⁻⁶(面法線がX軸とほぼ平行)のときは **グローバル+Y** を使う
4. **y軸** = z × x(右手系)

### 生成時点の幾何(順序安定性)

フレームは参照面の**providesされた時点**(フィーチャー生成直後)の重心・法線から
導出する。前送り後の現在形状(穴などで切り欠かれた面)の重心では**ない**。

理由: 現在形状の重心を使うと、先行フィーチャーが参照面を切り欠くたびに後続の
`center()` 配置が動いてしまい、フィーチャーの追加が既存配置を黙って変える。
生成時点の幾何に固定することで、配置は参照面の定義のみに依存し順序安定になる。
帰結として、参照面が後続操作で消滅していても配置面(平面)としては参照可能である。

また意味論としても、`center()` の設計意図は「**外形輪郭の中心**」であって
「残材(穴あき後)の重心」ではない(2026-07-12レビュー確認)。

例: ブロック天面(z=+Z)では x=+X, y=+Y。ブロック側面(z=+X)では基準軸が
切り替わり x = +Y の面内射影、y = z × x。

## ルートフィーチャーのグローバル配置(M2-2)

§4.0「グローバル座標の直書きはルートのみ許可」の実体として、ルートフィーチャー
(Block / Cylinder)は `at: Origin` に加えて **`Offset(from: Origin, d: (...))` による
グローバル平行移動**を受け付ける(M3のmate解決までの2体フィクスチャ配置にも使う)。
回転を伴うルート配置は未対応(ワールド軸限定)。

## Pos2

- `center()`: 原点 = フレーム原点(面の重心)
- `xy(u, v)`: 原点 = 重心 + u·x + v·y
- `from_edge(<edges>, d, along)`(M1-3実装済み): エッジ選択は**ちょうど1本**に
  定まること(複数はエラー — edges_between で特定する)。
  - M = エッジ中点、u = エッジ方向を**+X/+Y/+Zの順で最初に非直交な軸と正の内積**を
    持つ向きに正規化(エッジの内部向きに依存しない決定的向き付け)
  - w = z × u を参照面の生成時点重心の側に向けた面内直交方向
  - 原点 = M + along·u + d·w

## Offset

`offset(<placement>, (dx, dy, dz))` の (dx, dy, dz) は **fromフレームのローカル
(x, y, z)軸成分**である(グローバル座標ではない)。軸はfromフレームのまま。

## フィーチャーの向き規約

- **Hole / Pocket**(切削): 工具は −z(掘り込み)方向に生成される。
  Holeの `axis` providesの方向は掘り込み方向(−z)
- **Boss / 面上のCylinder**(付加): +z(外向き)方向に押し出される
- 工具は数値堅牢性のため表面から z 方向に 0.5〜1mm 突き抜けて生成される
  (貫通穴はバウンディングボックス対角+1mmのオーバーシュート)。
  この量は結果形状に影響しない(ブーリアンで消える)

## 未対応(M1-4時点)

- 非ルート配置の `Block`(直方体を面フレームに回転配置)
- 曲面上の配置(円筒面等) — フレーム導出が平面前提。必要になった時点で
  本書に規則を追加してから実装する
