# ADR-003: チェッカーは純関数、3値判定+margin+修復可能Evidenceの固定契約

**Status:** Accepted

## Context

LLMの生成は確率的であり、保証は決定的検証器が与える(提案は確率的、判定は決定的)。エージェントの修復ループを成立させるには、検証結果が (1) 機械可読、(2) 再現可能、(3) 修復材料として十分、である必要がある。

## Decision

1. **全チェッカーは共通トレイトを実装する純関数。** グローバル状態・乱数・時刻に依存しない。並列実行可能
2. **判定は3値: Pass / Fail / Inconclusive。** チェック不能(束縛失敗、メッシュ失敗、材料未定義)はFailと厳密に区別する。exit code 0/1/2
3. **Passでもmarginを必ず返す。** measured / threshold / margin を全結果に含め、余裕率をエージェントの目的関数・人間のレビュー指標にする
4. **Evidenceは修復可能な粒度。** 「干渉あり」ではなく「anchor_a, anchor_b, 最小距離, 最近接点座標」。Evidence単体でLLMが違反箇所を特定できることを受入基準(US-16)とする
5. **出力はJSON Lines固定**(`results.jsonl`)。人間向け表示は`adc report`が別途整形する
6. **結果はキャッシュ可能。** キーは hash(Part定義正規形 + Checker設定 + ADCバージョン)。SHA-256

## Consequences

- (+) CI・エージェントループ・人間レビューが同一の出力を消費する
- (+) 決定性テスト(バイト再現)が自動化できる
- (−) 近似手法のチェッカー(肉厚レイキャスト等)はサンプリング密度をChecker設定に含め、キャッシュキーに反映する規律が必要
- (−) marginの定義はチェッカーごとに文書化必須((measured−threshold)/threshold を基本形とする)
