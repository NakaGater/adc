# ADR-004: 未確定パラメータは第一級。3点ビルド+二分探索で区間を絞る

**Status:** Accepted

## Context

従来CADは全寸法の即時確定を強制するが、実務の構想設計は「この壁厚は2〜4mmで未決、DFM検証後に確定」という未確定状態を大量に含む。未確定を表現できないことが、早すぎる意思決定と根拠なき数値の温床になる。

## Decision

1. **Paramは `Determined(f64) | Open { range, nominal }` の直和型。** Openのまま build / check / export(公称値)可能
2. **Open含みの検証は3点評価:** 公称+区間両端でビルド&チェックし、全点Passで区間Pass。厳密な区間保証ではなく標本保証であることをdocに明記(単調性を仮定できるチェッカーが大半のため実用上十分)
3. **片端Failのとき二分探索で実行可能区間を推定**し、`suggested_range` として返す(`adc check --narrow`、反復上限デフォルト8)
4. **複数Openパラメータの同時絞り込みはMVPでは直積しない。** 1変数ずつ(他は公称固定)。多変数最適化はPhase 2のエージェント側の仕事とする

## Consequences

- (+) 「決めなくていいことを決めない」を機械が支援する。rationale: Assumption と組み合わせ、仮決めと確定が型で区別される
- (+) エージェントの設計空間探索がスキーマレベルでサポートされる
- (−) 3点評価はビルド回数を最大3倍にする。キャッシュ(ADR-003)とOpenパラメータ数の実務的節度で吸収
- (−) 非単調な応答(共振等)は標本保証の穴。MVPのチェッカー群(幾何・質量・代数)はほぼ単調であり許容。将来のFEM系チェッカー導入時に再検討
