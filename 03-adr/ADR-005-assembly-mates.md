# ADR-005: アセンブリは本格拘束ソルバーを持たず、mate列の逐次解決とする

**Status:** Accepted

## Context

MVPスコープは2〜10部品の小規模アセンブリ。汎用の幾何拘束ソルバー(連立非線形、自由度管理)は実装コストが非常に高く、SolveSpace級の資産が必要になる。一方、実務の小規模Assyの大半は「基準部品に対して部品を順に位置決めする」木構造で表現できる。

## Decision

1. **mateはアンカー参照で宣言する:** `coaxial(a.anchor, b.anchor)` / `coincident` / `distance(d)` / `angle(θ)`。部品再生成後もアンカー再束縛(ADR-001)によりmateが生き残る
2. **解決は逐次的:** 部品はDAG順に、先行部品に対するmate列から剛体変換を合成して確定する。連立ソルバーは持たない
3. **拘束不足は許容し、残自由度を報告する**(構想段階では未拘束が正常)。解決不能・矛盾は E-MATE-UNSOLVED{mate_id, 原因}
4. **循環参照はエラー。** mateグラフはDAGであることを静的検証する

## Consequences

- (+) 実装が数百行オーダーに収まり、決定的で高速。エラーが局所化される
- (+) 「どのmateが位置を決めたか」が説明可能(`adc explain`)で、rationale連鎖に乗る
- (−) 閉ループ拘束(リンク機構等)は表現不能。スコープ外と明記し、必要になった時点でソルバー統合をPhase 3のADRとして起票
- (−) DAG順序は宣言順に依存。順序非依存化(位相ソート)はM3内で実装する
