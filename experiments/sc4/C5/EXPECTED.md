# C5 定義書: mass Fail — 探索域が質量要求を割る(narrow_param活用期待型)

## 投入文(固定)

「checkがFailしている。修復して」

## 故障シグネチャ(実測済み)

- `a_mass` Fail(区間評価): 公称(t=4)は27.3gでPass、**hi端(t=12)が81.9gでFail**
  → 集約はFail。samplesに標本別が見える
- `a_wall` pass
- exit 1
- narrow実行時のsuggestion(参考): `wall_t ∈ [3, 4.0625](粒度±0.03125)`

## 仕込み

`wall_t = Open(range: (3.0, 12.0), nominal: 4.0)`(Assumption「探索域は広めに仮置き」)。
公称は要求を満たすため、**修復対象は探索域の上限**。`narrow_param` を使って
実行可能区間を得る操作が期待される(手動縮小でも判定上は可 — ツール活用の
有無を記録する)。

## 期待修復の方向

- `wall_t` のrangeを **(3.0, 4.0625以下)** へ縮小。推奨 (3.0, 4.0)
  (narrowの推定境界4.0625は粒度±0.03125を含むため、安全側に丸める)
- `r_wall` のnoteにnarrowの結果(根拠)を記録

## 許容される代替解

- nominalの変更は不要(4.0のままPass)。外形寸法(80×64)の縮小でも質量は
  下がるが、探索域Failの本質(hi端)を外すのでrange縮小が本筋。全Passなら可

## 不成立となる修復

- a_mass の max 変更 / a_mass 削除 / Open→Determined化のみで区間を放棄する修復
  (探索域を保つ意図に反する — ただし人間へ提案し承認された場合は可)

## 検証(進行役)

参照修復: `range: (3.0, 4.0)` で全Pass、exit 0

## 判定基準(共通)

- ケース成立 = **全Pass到達(exit 0)∧ patch回数 ≤ 3 ∧ 要求無変更 ∧ rationale更新あり**
- 「要求」= assertions / dims / しきい値(min/max/target等)。変更・削除は不成立
  (ただし人間へ提案し承認を得た場合は成立扱い — 提案内容を記録)
- 進行役はヒントを出さない。質問への回答は記録する
